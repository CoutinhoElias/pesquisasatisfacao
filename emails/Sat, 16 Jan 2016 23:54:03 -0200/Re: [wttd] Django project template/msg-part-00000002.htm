Henrique Nogueira parabéns pela iniciativa, mas sugiro que você deixe o seu README.md mais completo.

Escreva as instruções para instalação completa, semelhante ao que o Henrique ensina no curso.

Veja meu exemplo: https://github.com/rg3915/wttd2#como-desenvolver

No seu caso, segue o comando abaixo corrigido (faltou uma \), e sugiro que já deixe um nome de projeto pré definido. Naturalmente alguém vai copiar o comando na íntegra.

django-admin.py startproject \
  --template=https://github.com/henriquenogueira/django_project_template/zipball/master \
  --extension=py,rst,yml \
  --name=Procfile \
  myproject .

Antes disso falta:

* criar o virtualenv
* ativar o virtualenv
* instalar tudo com pip
* copiar o env-sample

pip install -r requirements/dev.txt
cp contrib/env-sample .env

Pessoal,

Nos últimos dias, para fins de prática e estudo, precisei criar e recomeçar várias idéias do zero. Percebi que o processo inicial de configurar um projeto (com dj-static, python-decouple, etc) é um pouco árduo. Por isso, criei um template de projeto.

O django-admin startproject, quando executado, cria a estrutura padrão de um projeto django, mas isso pode ser sobrescrito. 

Criei o repositório https://github.com/henriquenogueira/django_project_template. Lá tem instruções básicas de como criar um projeto com a cara do Eventex. 

Vale ressaltar que pull requests serão enormemente bem-vindos :)
