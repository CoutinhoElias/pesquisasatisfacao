Pelo o que eu entendi do diagrama de sequencia apresentado pelo Henrique no M2A04 nós podemos substituir o fornecimento dos arquivos estaticos do djstatic por um outro http server qualquer, como http.server do proprio python, amazon s3 ou whatever.

Até então tudo tranquilo, realizei a criação do bucket s3 na amazon e realizei o upload dos arquivos para lá manualmente.
Após a alteração do STATIC_URL para a nova url do bucket s3 do amazon o ambiente de dev carregou normalmente assim como quando realizei os testes com o módulo http.server do python.

A dúvida que ficou no entanto é a seguinte:

Toda vez que o designer enviar novos arquivos ou hover alterações eu teria que realizar o collect static e então fazer upload dos arquivos para o bucket s3 da amazon manualmente ? Vejo esta como uma opção, mas se tratando de python e django acredito que não seja a melhor.

Então penso.. há algum módulo no django que interaja com o bucket s3 utilizando as credenciais da amazon para realizar alterações no bucket sempre que necessário e assim isso ficaria automatizado toda fez que for feito deploy da aplicação ? Qual seria este cara ?
