Em 2 de fevereiro de 2016 01:26, severo.fabricio <severo.fabricio@gmail.com> escreveu:
A um tempo atrás estava fazendo uns "estudos" de django, e, para facilitar minha vida, criei o seguinte "script" que coloquei no ~/.bash_aliases

deploy () {
  heroku maintenance:on;
  git push heroku master;
  heroku run python manage.py migrate;
  heroku maintenance:off;
}

ele põe o heroku em modo manutenção antes de realizar o push. Desta forma, qualquer pessoa que tentar acessar o site enquanto o sistema está sendo atualizado vai receber uma mensagem de manutenção (não verá o site).
Isso é útil no caso das migrações, quando é "perigoso" deixar o sistema rodando antes de aplicar elas.

Compartilhando caso seja útil para alguém.
