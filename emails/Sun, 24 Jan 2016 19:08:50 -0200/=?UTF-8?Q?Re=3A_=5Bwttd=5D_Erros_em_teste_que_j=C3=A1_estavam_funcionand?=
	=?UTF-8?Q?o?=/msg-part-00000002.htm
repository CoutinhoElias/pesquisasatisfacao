Régis,

Acho que duas abordagens pro CPF são válidas:

1) Tratar o CPF como numérico, armazenando-o num campo com o tipo apropriado, formatando quando precisar.
2) Tratar o CPF como string, armazenando-o num campo string já formatado, usando-o as-is.

Precisa alinhar com o contexto do seu problema, pra ver qual abordagem é mais apropriada. Geralmente é a 2.

Quanto ao seu código:

        ...
        cpf = cpf[:3] + cpf[4:7] + cpf[8:11] + cpf[12:]
        ...

Lembre-se que strings são imutáveis, e quando vc concatena elas está criando outras strings na memória.

No caso, são criadas uma pra cada slice, e uma pra cada soma, se não estiver enganado... Seriam 7 ao total.

Prefiro:

    def clean_cpf(self):
        return = self.cleaned_data['cpf'].replace('.','').replace('-','')

Ah, colocaria um teste pra saber se o cleaned_data está vazio ou nulo. ;)

    def clean_cpf(self):
        return = self.cleaned_data['cpf'].replace('.','').replace('-','') if self.cleaned_data['cpf'] else self.cleaned_data['cpf']
